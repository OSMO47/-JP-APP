# JLPT Vocabulary Trainer

เว็บแอปสำหรับฝึกคำศัพท์ภาษาญี่ปุ่นรองรับ JLPT ระดับ N5, N4 และ N3 พัฒนาด้วย Next.js และ Tailwind CSS

## คุณสมบัติหลัก

- หน้าหลักเลือกโหมดและระดับ JLPT ที่ต้องการฝึก
- โหมดสุ่มคำศัพท์พร้อมช้อยส์ 4 ตัวเลือก และสถิติการทำแบบทดสอบ
- เลือกฝึกตามหมวดคำศัพท์ย่อยของ JLPT N5 ที่จัดกลุ่มไว้เป็นชุดหัวข้อ
- โหมดคลังคำศัพท์ของฉัน สำหรับบันทึกคำศัพท์เพิ่มเติมลงใน Local Storage
- รองรับการใช้งานทั้งภาษาไทยและภาษาญี่ปุ่น พร้อมดีไซน์ที่ตอบสนองทุกอุปกรณ์

## การเริ่มต้นพัฒนา

1. ติดตั้งแพ็กเกจที่จำเป็น

   ```bash
   npm install
   ```

2. เริ่มโหมดพัฒนา

   ```bash
   npm run dev
   ```

3. เปิดเบราว์เซอร์ที่ `http://localhost:3000`

## โครงสร้างโฟลเดอร์สำคัญ

```
/pages
  index.js        # หน้าเลือกระดับและโหมด
  quiz.js         # โหมดสุ่มคำศัพท์
  my-vocab.js     # คลังคำศัพท์ของฉัน
/components
  WordCard.js     # การ์ดแสดงคำศัพท์
  QuizOption.js   # ปุ่มตัวเลือกคำตอบ
/data
  vocab_n5.json
  vocab_n4.json
  vocab_n3.json
```

### โครงสร้างข้อมูลคำศัพท์

ตั้งแต่เวอร์ชันนี้เป็นต้นไป ไฟล์คำศัพท์รองรับทั้งรูปแบบ **ชุดคำศัพท์ (vocabulary sets)** และ **รายการคำแบบเดิม** เพื่อให้จัดหมวดหมู่ได้สะดวกและต่อยอดง่าย ตัวอย่างข้อมูลสำหรับระดับ N5:

```jsonc
[
  {
    "set": 1,
    "title": "พื้นฐานทั่วไป",
    "_comment": "คำศัพท์ทั่วไปที่ใช้ในชีวิตประจำวัน",
    "words": [
      { "word": "学校", "reading": "がっこう", "meaning": "โรงเรียน" },
      { "word": "先生", "reading": "せんせい", "meaning": "ครู" }
    ]
  },
  {
    "set": 2,
    "title": "ชีวิตประจำวัน",
    "words": [
      { "word": "電話", "reading": "でんわ", "meaning": "โทรศัพท์" }
    ]
  }
]
```

แต่ละคำสามารถเพิ่มฟิลด์เสริมได้ เช่น `english`, `romaji`, `pos`, `example` (มี `ja`, `kana`, `thai`) ระบบจะเลือกใช้เท่าที่มีเพื่อแสดงรายละเอียดหลังตอบคำถาม

สำหรับระดับที่ยังเป็นรูปแบบอาร์เรย์ของคำศัพท์ (เช่น N4/N3 เดิม) ระบบก็ยังอ่านได้ตามปกติ โดยใช้คีย์ที่มีอยู่ เช่น `kanji`, `kana`, `thai` เป็นต้น

## การดีพลอย

### ตัวเลือกที่ 1: Vercel (แนะนำสำหรับ Next.js)

1. สร้างบัญชีหรือเข้าสู่ระบบที่ [https://vercel.com](https://vercel.com).
2. คลิก **New Project** แล้วเชื่อมต่อกับ Git provider (GitHub/GitLab/Bitbucket) ที่เก็บซอร์สโค้ดนี้ไว้
   - หากยังไม่ได้ push โปรเจกต์ ให้สร้าง repository ใหม่และ push ด้วยคำสั่ง `git remote add origin ...` และ `git push -u origin main` ก่อน
3. เลือก repository ที่ต้องการดีพลอย แล้วตรวจสอบค่าเริ่มต้น
   - **Framework Preset:** ควรถูกตั้งเป็น Next.js อัตโนมัติ
   - **Build Command:** `npm run build`
   - **Output Directory:** `.next`
4. กด **Deploy** และรอให้ build เสร็จ ระบบจะให้โดเมนพร้อมใช้งาน เช่น `https://your-project.vercel.app`
5. ทุกครั้งที่ push ไปยัง branch `main` (หรือ branch ที่ตั้งค่าไว้) Vercel จะ build & deploy ให้โดยอัตโนมัติ

> หากต้องการตั้งค่าตัวแปรสภาพแวดล้อม (Environment Variables) ให้ไปที่ **Settings → Environment Variables** ของโปรเจกต์ใน Vercel แล้วเพิ่ม key/value ตามต้องการก่อนกด Redeploy

### ตัวเลือกที่ 2: รันบนเซิร์ฟเวอร์ Node.js ของคุณเอง

1. ติดตั้ง dependencies และ build โปรเจกต์บนเครื่องเซิร์ฟเวอร์

   ```bash
   npm install
   npm run build
   ```

2. เริ่มเซิร์ฟเวอร์ production ด้วยคำสั่ง

   ```bash
   npm run start
   ```

3. ตั้งค่า process manager เช่น **PM2**, **forever**, หรือ systemd เพื่อให้แอปทำงานตลอดเวลา ตัวอย่าง PM2:

   ```bash
   npm install -g pm2
   pm2 start npm --name jlpt-app -- run start
   pm2 save
   ```

4. ตั้งค่า reverse proxy (เช่น Nginx หรือ Caddy) เพื่อชี้โดเมนของคุณไปยังพอร์ตที่แอปรันอยู่ (ค่าเริ่มต้นคือ `3000`).

5. เปิดใช้งาน HTTPS ด้วย Let's Encrypt หรือบริการ SSL ที่คุณใช้งานอยู่เพื่อความปลอดภัย

### ตัวเลือกที่ 3: Docker (ทางเลือกเพิ่มเติม)

1. สร้างไฟล์ `Dockerfile` เช่นตัวอย่างด้านล่าง:

   ```dockerfile
   FROM node:18-alpine
   WORKDIR /app
   COPY package*.json ./
   RUN npm install --production
   COPY . .
   RUN npm run build
   EXPOSE 3000
   CMD ["npm", "run", "start"]
   ```

2. สร้างอิมเมจและรันคอนเทนเนอร์

   ```bash
   docker build -t jlpt-app .
   docker run -d -p 3000:3000 --name jlpt jlpt-app
   ```

3. ตั้งค่า reverse proxy/SSL เช่นเดียวกับตัวเลือก Node.js ปกติ

## หลังจากใช้งานหรือพัฒนาเสร็จควรทำอะไรต่อ?

- **ทดสอบโหมดต่าง ๆ** — เปิดหน้า Home แล้วลองเข้าโหมด Quiz และ My Vocabulary เพื่อยืนยันว่าข้อมูลโหลดได้ถูกต้องและการบันทึกคำศัพท์ใน Local Storage ทำงานปกติ
- **เพิ่มคำศัพท์ของคุณเอง** — ใช้ปุ่ม "เพิ่มคำศัพท์" ในหน้า My Vocabulary เพื่อเติมฐานข้อมูลส่วนตัว หากต้องการเพิ่มไฟล์คำศัพท์เริ่มต้นให้แก้ไขในโฟลเดอร์ `data`
- **บันทึกสถิติการเรียนรู้** — หลังทำแบบทดสอบ ให้จดบันทึกผลลัพธ์หรือถ่ายภาพหน้าจอเพื่อเก็บสถิติการเรียนด้วยตนเอง (หรือขยายระบบคะแนนเพิ่มเติมในอนาคต)
- **ปรับแต่งสไตล์** — แก้ไขไฟล์ `styles/globals.css` หรือค่าธีมใน `tailwind.config.js` เพื่อให้เหมาะกับแบรนด์หรือโทนสีที่ต้องการ
- **ดีพลอยสู่ Production** — เมื่อพร้อมเผยแพร่ ให้รัน `npm run build` และดีพลอยผ่านบริการอย่าง Vercel, Netlify หรือโฮสต์ Node.js อื่น ๆ
